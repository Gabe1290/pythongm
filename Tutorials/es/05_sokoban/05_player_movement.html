Paso 4: Movimiento del Jugador

El jugador se mueve en pasos fijos de 32 píxeles (movimiento basado en cuadrícula).

Importante: En Sokoban, el movimiento debe verificar:
1. ¿Hay una pared? (No puede moverse)
2. ¿Hay una caja? (¿Podemos empujarla?)
3. ¿Hay una pared/caja detrás de la caja? (No puede empujar)

Mover a la Derecha (Flecha Derecha)

En obj_player:

  1. Agregar Evento: Teclado > Presionar Derecha
  2. Agregar Acción: Control > Ejecutar Código
  3. Ingresa este código:
// Verificar si podemos movernos a la derecha
if (!place_meeting(x + 32, y, obj_wall)) {
    var crate = instance_place(x + 32, y, obj_crate);
    if (crate != noone) {
        // Hay una caja - ¿podemos empujarla?
        if (!place_meeting(x + 64, y, obj_wall) &&
            !place_meeting(x + 64, y, obj_crate)) {
            crate.x += 32;
            x += 32;
        }
    } else {
        // No hay caja, solo moverse
        x += 32;
    }
}

Mover a la Izquierda (Flecha Izquierda)

Mismo código pero cambia:

  • + 32 se convierte en - 32
  • + 64 se convierte en - 64

Mover Arriba (Flecha Arriba)

Mismo código pero usa y en lugar de x:

  • x + 32 se convierte en y - 32
  • x + 64 se convierte en y - 64
  • crate.x se convierte en crate.y

Mover Abajo (Flecha Abajo)

Igual que Arriba pero con + 32 y + 64.

Haz clic en "Siguiente" para agregar la detección de condición de victoria.
