Step 4: Player Movement (Step Event)

This is the core of our platformer: movement, gravity, jumping, and collision.

Add Step Event

In obj_player:

  1. Add Event: Step > Step
  2. Add Action: Control > Execute Code
// === HORIZONTAL MOVEMENT ===
var move_input = keyboard_check(vk_right) - keyboard_check(vk_left);
hsp = move_input * hspeed_max;

// === GRAVITY ===
vsp += gravity_force;
if (vsp > vspeed_max) vsp = vspeed_max;

// === GROUND CHECK ===
on_ground = place_meeting(x, y + 1, obj_ground);

// === JUMPING ===
if (on_ground && (keyboard_check_pressed(vk_up) || keyboard_check_pressed(vk_space))) {
    vsp = jump_force;
}

// === HORIZONTAL COLLISION ===
if (place_meeting(x + hsp, y, obj_ground)) {
    while (!place_meeting(x + sign(hsp), y, obj_ground)) {
        x += sign(hsp);
    }
    hsp = 0;
}
x += hsp;

// === VERTICAL COLLISION ===
if (place_meeting(x, y + vsp, obj_ground)) {
    while (!place_meeting(x, y + sign(vsp), obj_ground)) {
        y += sign(vsp);
    }
    vsp = 0;
}
y += vsp;

How It Works

  1. Input: Right arrow = 1, Left = -1, None = 0
  2. Gravity: Adds to vsp each frame (falling faster)
  3. Ground Check: Looks 1 pixel below for ground
  4. Jump: Only works when on_ground is true
  5. Collision: Move pixel-by-pixel until hitting wall
The Magic: The while loop moves us pixel-by-pixel to the edge of the wall, preventing us from getting stuck!
