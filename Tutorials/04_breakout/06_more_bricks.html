Phase 3: More Brick Rows

Let's add three more rows of colorful bricks! Thanks to the parent object, we don't need to change the ball's code at all — each new brick color automatically inherits the collision behavior.

Create the New Brick Sprites

Create three new sprites. For each one, draw a solid rectangle or import the sample image:

SpriteSizeColorSample file
spr_brick_orange32×16Orangespr_brick_orange.png
spr_brick_yellow32×16Yellowspr_brick_yellow.png
spr_brick_green32×16Greenspr_brick_green.png

Option A — Draw them yourself

For each sprite, open the Sprite Editor and draw a solid rectangle in the matching color. Size: 32×16 pixels.

Option B — Load the sample sprites

For each sprite, right-click → Import Image... and select the matching file from Tutorials/04_breakout/assets/.

Create the New Brick Objects

Each brick uses the same parent, so the ball's collision code works for all of them.

For each new brick color:

  1. Create the object with the name and sprite below
  2. Set Parent to obj_brick_parent
  3. Check Solid
ObjectSpriteParent
obj_brick_orangespr_brick_orangeobj_brick_parent
obj_brick_yellowspr_brick_yellowobj_brick_parent
obj_brick_greenspr_brick_greenobj_brick_parent
No programming needed! Because all bricks share the same parent (obj_brick_parent), the ball already knows how to bounce off them and destroy them. That's the power of parent objects!

Add Brick Rows to the Room

  1. Open room_breakout
  2. Arrange the bricks in 4 rows near the top of the room:

  • Top row: Red bricks (obj_brick_red)
  • Second row: Orange bricks (obj_brick_orange)
  • Third row: Yellow bricks (obj_brick_yellow)
  • Fourth row: Green bricks (obj_brick_green)

Use Snap to Grid (32×16) for easy alignment.

Tip: Keep brick sizes consistent (32×16) so they line up nicely in rows!

Test Your Game!

  • Each brick gives 10 points
  • Try to clear all the rows!
