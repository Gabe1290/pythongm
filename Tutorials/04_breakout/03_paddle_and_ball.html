Phase 1: Objects, Paddle and Ball

Let's create the game objects and program them so we can test the game quickly!

Create the Objects

We need two different wall types so the ball knows which direction to bounce.

  1. Right-click on Objects → Create New Object
  2. Create obj_wall_side — sprite: spr_wall, check Solid
  3. Create obj_wall_top — sprite: spr_wall, check Solid
  4. Create obj_paddle — sprite: spr_paddle, check Solid
  5. Create obj_ball — sprite: spr_ball, leave Solid unchecked
  6. Create obj_death_zone — sprite: spr_death_zone, uncheck Visible
Why two wall types? When the ball hits a side wall, it reverses its horizontal direction. When it hits the top wall, it reverses its vertical direction. Using two objects lets us program different bounce behaviors!

Program the Paddle

  1. Double-click obj_paddle to open the Object Editor
  2. Click the Blockly tab
How Blockly works: Drag blocks from the toolbox on the left into the workspace. Snap action blocks inside event blocks to define what happens. When you're done, click Apply to Events to save your blocks.

Move Left

Keyboard: Left (held)
Set horizontal speed to -8

Find Keyboard (held) in Events, select Left. Then snap Set Horizontal Speed from Movement inside. Type -8 (negative = left).

Move Right

Keyboard: Right (held)
Set horizontal speed to 8

Same as above, but choose Right and type 8 (positive = right).

Stop When No Key Pressed

Keyboard: No key
Stop movement

This prevents the paddle from sliding forever when you release the key.

Stop at Side Walls

When colliding with obj_wall_side
Stop movement

This keeps the paddle inside the play area.

You should have 4 event blocks. Click Apply to Events to save.

Program the Ball

  1. Double-click obj_ball to open the Object Editor
  2. Click the Blockly tab

Start the Ball Moving

Create Event
Set horizontal speed to 3
Set vertical speed to -3

The Create Event runs once when the game starts. Setting both speeds makes the ball move diagonally upward.

Why two speeds? Horizontal speed (3) moves the ball sideways. Vertical speed (-3) moves it upward (negative = up). Together, the ball moves diagonally!

Bounce Off Side Walls

When colliding with obj_wall_side
Reverse horizontal movement

The ball reverses its left/right direction when hitting a side wall.

Bounce Off Top Wall

When colliding with obj_wall_top
Reverse vertical movement

The ball bounces downward when hitting the top wall.

Bounce Off Paddle

When colliding with obj_paddle
Reverse vertical movement

Same as the top wall — the ball bounces back upward.

Ball Falls Off Screen

When colliding with obj_death_zone
Jump to x: 320 y: 100

When the ball hits the death zone at the bottom, it respawns above the paddle. We'll add lives later — for now, the ball just comes back!

You should have 5 event blocks. Click Apply to Events to save.

How bouncing works: The ball always moves diagonally. When it hits a vertical surface (side wall), we reverse horizontal movement. When it hits a horizontal surface (top wall, paddle), we reverse vertical movement. This creates realistic bouncing!
