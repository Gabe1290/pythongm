Step 2: Create the Objects

Now we'll create all the game objects. We need two different wall types so the ball knows which direction to bounce!

Create the Side Wall Object

  1. Right-click on Objects and select Create Object
  2. Name it obj_wall_side
  3. Set the sprite to spr_wall
  4. Check "Solid"
  5. No events needed

Create the Top Wall Object

  1. Create object obj_wall_top
  2. Set the sprite to spr_wall
  3. Check "Solid"
  4. No events needed
Why two wall types? When the ball hits a side wall, it reverses its horizontal direction. When it hits the top wall, it reverses its vertical direction. Using two objects lets us program different bounce behaviors!

Create the Parent Brick Object

We'll use a parent object so all bricks share the same collision behavior.

  1. Create object obj_brick_parent
  2. No sprite needed (it's just a template)
  3. Check "Solid"

Create the Colored Brick Objects

For each brick color:

  1. Create obj_brick_red with spr_brick_red
  2. Set Parent to obj_brick_parent
  3. Check "Solid"

Repeat for obj_brick_orange, obj_brick_yellow, and obj_brick_green.

Create the Paddle Object

  1. Create object obj_paddle
  2. Set sprite to spr_paddle
  3. Check "Solid"

Create the Ball Object

  1. Create object obj_ball
  2. Set sprite to spr_ball
  3. Leave "Solid" unchecked

Create the Death Zone Object

  1. Create object obj_death_zone
  2. Set sprite to spr_death_zone
  3. Uncheck "Visible" — it should be invisible

Create the Game Controller

  1. Create object obj_game_controller
  2. No sprite needed (leave it empty)
  3. Leave "Solid" unchecked
Parent Objects: When the ball collides with obj_brick_parent, it will detect collisions with ALL brick types! This saves us from writing the same collision code four times.
