Step 4: Player Movement

The player moves in fixed 32-pixel steps (grid-based movement).

Important: In Sokoban, movement must check:
1. Is there a wall? (Can't move)
2. Is there a crate? (Can we push it?)
3. Is there a wall/crate behind the crate? (Can't push)

Move Right (Right Arrow)

In obj_player:

  1. Add Event: Keyboard > Press Right
  2. Add Action: Control > Execute Code
  3. Enter this code:
// Check if we can move right
if (!place_meeting(x + 32, y, obj_wall)) {
    var crate = instance_place(x + 32, y, obj_crate);
    if (crate != noone) {
        // There's a crate - can we push it?
        if (!place_meeting(x + 64, y, obj_wall) &&
            !place_meeting(x + 64, y, obj_crate)) {
            crate.x += 32;
            x += 32;
        }
    } else {
        // No crate, just move
        x += 32;
    }
}

Move Left (Left Arrow)

Same code but change:

  • + 32 becomes - 32
  • + 64 becomes - 64

Move Up (Up Arrow)

Same code but use y instead of x:

  • x + 32 becomes y - 32
  • x + 64 becomes y - 64
  • crate.x becomes crate.y

Move Down (Down Arrow)

Same as Up but with + 32 and + 64.
