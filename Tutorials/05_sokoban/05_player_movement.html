Step 4: Player Movement (Blockly)

The player moves in fixed 32-pixel steps (grid-based movement). We'll use Blockly visual blocks to program this.

Important: In Sokoban, movement must check:
1. Is there a wall? (Can't move)
2. Is there a crate? (Can we push it?)
3. Is there a wall/crate behind the crate? (Can't push)

Open the Blockly Editor

  1. Double-click obj_player in the Asset Tree to open the Object Editor
  2. Click the Blockly tab at the top of the editor

Move Right (Right Arrow)

Drag these blocks from the toolbox and snap them together:

Keyboard Press: Right
Move Grid: Right, 32

Find Keyboard Press in the Events category and select Right. Then find Move Grid in the Movement category, set direction to Right and grid size to 32.

Move Left, Up, Down

Repeat for the other three directions:

Keyboard Press: Left
Move Grid: Left, 32
Keyboard Press: Up
Move Grid: Up, 32
Keyboard Press: Down
Move Grid: Down, 32

Each Keyboard Press event fires once per key press — perfect for grid movement!

Push Crates (Collision with obj_crate)

When colliding with obj_crate
If Can Push facing
Push Other Instance 32

Find Collision in Events and type obj_crate. Then find If Can Push in Control — it checks whether the crate can be pushed in the direction the player is moving. Inside the "then" section, snap a Push Other Instance block with distance 32.

Stop at Walls (Collision with obj_wall)

When colliding with obj_wall
Stop movement

Find Collision in Events and type obj_wall. Snap a Stop Movement block inside. This prevents the player from walking through walls.

Apply Your Blocks

  1. You should have 6 event blocks in your workspace (4 keyboard + 2 collision)
  2. Click the Apply to Events button to save
How it works: Pressing an arrow key moves the player exactly 32 pixels (one grid cell). If the player hits a crate, the If Can Push block checks whether the space behind the crate is empty. If it is, both the player and crate move. Walls block all movement.
