Schritt 4: Spielerbewegung

Der Spieler bewegt sich in festen 32-Pixel-Schritten (rasterbasierte Bewegung).

Wichtig: Bei Sokoban muss die Bewegung pruefen:
1. Ist dort eine Wand? (Kann nicht bewegen)
2. Ist dort eine Kiste? (Koennen wir sie schieben?)
3. Ist eine Wand/Kiste hinter der Kiste? (Kann nicht schieben)

Nach rechts bewegen (Pfeiltaste Rechts)

In obj_player:

  1. Event hinzufuegen: Keyboard > Press Right
  2. Aktion hinzufuegen: Control > Execute Code
  3. Diesen Code eingeben:
// Pruefen, ob wir nach rechts bewegen koennen
if (!place_meeting(x + 32, y, obj_wall)) {
    var crate = instance_place(x + 32, y, obj_crate);
    if (crate != noone) {
        // Da ist eine Kiste - koennen wir sie schieben?
        if (!place_meeting(x + 64, y, obj_wall) &&
            !place_meeting(x + 64, y, obj_crate)) {
            crate.x += 32;
            x += 32;
        }
    } else {
        // Keine Kiste, einfach bewegen
        x += 32;
    }
}

Nach links bewegen (Pfeiltaste Links)

Gleicher Code, aber aendern:

  • + 32 wird zu - 32
  • + 64 wird zu - 64

Nach oben bewegen (Pfeiltaste Hoch)

Gleicher Code, aber y statt x verwenden:

  • x + 32 wird zu y - 32
  • x + 64 wird zu y - 64
  • crate.x wird zu crate.y

Nach unten bewegen (Pfeiltaste Runter)

Wie oben, aber mit + 32 und + 64.

Klicke auf "Weiter", um die Siegbedingung hinzuzufuegen.
