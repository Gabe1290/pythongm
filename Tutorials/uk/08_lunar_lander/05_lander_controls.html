Step 4: Lander Controls (Step Event)

Add gravity, thrust, horizontal movement, and boundary checks.

Add Step Event

In obj_lander:

  1. Add Event: Step > Step
  2. Add Action: Control > Execute Code
// Don't update if game is over
if (landed || crashed) exit;

// === GRAVITY ===
vsp += gravity_force;

// === THRUST (Up Arrow) ===
if (keyboard_check(vk_up) && fuel > 0) {
    vsp -= thrust_force;
    fuel -= fuel_use;
    if (fuel < 0) fuel = 0;
}

// === HORIZONTAL CONTROL ===
if (keyboard_check(vk_left)) hsp -= 0.05;
if (keyboard_check(vk_right)) hsp += 0.05;

// === LIMIT SPEED ===
hsp = clamp(hsp, -max_speed, max_speed);
vsp = clamp(vsp, -max_speed, max_speed);

// === APPLY MOVEMENT ===
x += hsp;
y += vsp;

// === SCREEN BOUNDARIES ===
if (x < 16) { x = 16; hsp = 0; }
if (x > room_width - 16) { x = room_width - 16; hsp = 0; }
if (y < 16) { y = 16; vsp = 0; }

How It Works

  1. Gravity: Adds to vsp every frame (falling)
  2. Thrust: Subtracts from vsp (rising), uses fuel
  3. Clamp: Limits speed to safe range
  4. Boundaries: Keeps lander on screen
Key: The clamp() function limits a value between min and max!
