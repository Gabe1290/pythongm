Phase 4: Winning the Game

Right now the player can lose, but there's no way to win. We'll fix that by checking after every step: "Are there any coins left?" — if not, we go to a brand-new win room.

Build the Win Room

Create the Win Sprite (Optional)

If you want a celebration look, create a sprite called spr_win_text. This step is optional — the text drawn in the next step works without a sprite.

Create the Win-Text Object

  1. Right-click Objects → Create New Object
  2. Name it obj_win_text
  3. Open the Blockly tab and add these two events:

Show the win message:

When drawing
Draw text YOU WIN! at x:412 y:320
Draw text Press SPACE to play again at x:340 y:400

Restart on space:

Key press: space
Restart Game

Click Apply to Events.

Create the Win Room

  1. Right-click Rooms → Create New Room
  2. Name it room_win
  3. Set the background color to a celebratory color (light green or gold)
  4. Place one obj_win_text instance anywhere in the room (its drawing is at fixed coordinates)

The Win Check

Now the important part: every step, the player checks if any coins are left. When the count reaches zero, we go to the win room.

  1. Open obj_player
  2. Click the Blockly tab
  3. Add this Step event:
Every step
If count of obj_coin == 0 then
Go to room room_win

Click Apply to Events.

What this does: The If Count Of block looks at the room and counts how many obj_coin instances are still alive. When you've caught them all, the count is zero, the comparison == 0 is true, and the Go to room action runs.

Test Your Full Game!

Press F5 to play!

  • Catch every coin → YOU WIN! screen appears
  • Touch the enemy → GAME OVER screen appears
  • Press SPACE from either ending to play again

You Built a Complete Game!

Congratulations — that's a full game loop with:

  • Player input (keyboard movement)
  • Two kinds of collision (good and bad)
  • A score the player can see grow
  • A clear win state and lose state
  • A way to restart and try again
Try this next: add more coins, place a second enemy, or change the coin and enemy speeds in their When created events. Every change is just a number you can edit and test.
