## Why it looked like a red and black grid

The 36 tiles were created and positioned correctly (a 6×6 grid of adjacent 32px squares), but everything that should have given them color was broken:

- All tiles shared the object default sprite `spr_red` — the blue/green/yellow sprites exist in the project but were never assigned to instances; color was supposed to come from a custom Draw event.
- That Draw event reads a per-tile `color` variable that the GridManager assigns randomly through the GML layer — and that assignment never stuck, so every tile stayed at its default `color = "red"`, drawn as a red fill plus black outline.
- The engine's fallback for a failed sprite is *also* a red box with a black border ([game_runner.py:541-545](runtime/game_runner.py#L541-L545)), so even total failure looks the same.
- The GML compat layer had crashes lying in wait regardless — e.g. it defines its own `abs()` and `round()` that call themselves recursively, so the first tile-swap click would have hit an infinite recursion.
