## 1. Введение

PyGameMaker — это образовательная среда разработки игр, вдохновлённая GameMaker. Она позволяет создавать 2D-игры визуально, используя систему событий и действий с перетаскиванием, без необходимости писать код. Она разработана с двумя целями:

- **Обучение разработке игр** через интуитивный визуальный интерфейс
- **Обучение программированию на Python** через открытый исходный код IDE

PyGameMaker использует PySide6 (Qt) для интерфейса IDE и Pygame для игрового движка. Игры можно экспортировать как автономные исполняемые файлы, мобильные приложения или HTML5-веб-игры.

### Основные возможности

- Визуальное программирование событий/действий (80+ встроенных действий)
- Редактор спрайтов с поддержкой анимации
- Редактор комнат с размещением экземпляров, слоями тайлов и прокруткой фона
- Предварительный просмотр игры в реальном времени по F5
- Экспорт в Windows EXE, мобильные платформы (Android/iOS через Kivy), HTML5 и роботы Thymio
- Интеграция с Google Blockly для визуальных блоков кода
- Симулятор образовательного робота Thymio
- Многоязычный интерфейс (8+ языков)
- Тёмная и светлая темы

## 2. Установка и настройка

### Требования

- Python 3.10 или выше
- PySide6
- Pygame
- Pillow (PIL)

### Установка зависимостей

```bash
pip install PySide6 pygame Pillow
```

### Дополнительные зависимости

Для функций экспорта:

```bash
pip install pyinstaller    # For EXE export
pip install kivy           # For mobile export
pip install buildozer      # For Android builds
pip install jinja2         # For code generation templates
```

### Запуск IDE

```bash
python main.py
```

Окно IDE открывается с размерами по умолчанию 1400x900 пикселей. Положение и размер окна сохраняются между сеансами.

## 3. Обзор IDE

### Расположение окон

IDE использует трёхпанельную компоновку:

```
+-------------------+---------------------------+------------------+
|   Дерево ресурсов |      Область редактора    |   Свойства       |
| (Левая панель)    | (Центральная панель)      | (Правая панель)  |
|                   |                           |                  |
|   - Спрайты       |   Редакторы с вкладками   |   Контекстно-    |
|   - Звуки         |   для спрайтов, объектов  |   зависимые      |
|   - Фоны          |   и комнат                |   свойства       |
|   - Объекты       |                           |                  |
|   - Комнаты       |                           |                  |
|   - Скрипты       |                           |                  |
|   - Шрифты        |                           |                  |
+-------------------+---------------------------+------------------+
|                       Строка состояния                           |
+------------------------------------------------------------------+
```

- **Левая панель (Дерево ресурсов):** Отображает все ресурсы проекта, организованные по типам. Дважды щёлкните ресурс, чтобы открыть его в редакторе.
- **Центральная панель (Область редактора):** Рабочее пространство с вкладками для редактирования спрайтов, объектов и комнат. По умолчанию отображается вкладка приветствия.
- **Правая панель (Свойства):** Показывает контекстно-зависимые свойства для текущего активного редактора. Может быть свёрнута.
- **Строка состояния:** Показывает текущую операцию и имя проекта.

- **Спрайты** — Изображения и анимационные полосы для игровых объектов

## 4. Работа с проектами

### Создание нового проекта

1. Выберите **Файл > Новый проект** (Ctrl+N)
2. Введите **Имя проекта**
3. Выберите **Расположение** для папки проекта
4. Выберите **Шаблон** (необязательно):
   - **Пустой проект** — Чистый проект
   - **Шаблон платформера** — Предварительно настроен для платформеров
   - **Шаблон вида сверху** — Предварительно настроен для игр с видом сверху
5. Нажмите **Создать проект**

### Структура проекта

```
MyProject/
├── project.json          # Main project file
├── sprites/              # Sprite images and metadata
│   ├── player.png
│   └── player.json
├── objects/              # Object definitions
│   └── obj_player.json
├── rooms/                # Room layout data
│   └── room_start.json
├── sounds/               # Audio files and metadata
├── backgrounds/          # Background images
└── thumbnails/           # Auto-generated previews
```

### Сохранение проектов

- **Ctrl+S** сохраняет проект в текущем расположении
- **Ctrl+Shift+S** сохраняет в новое расположение
- **Автосохранение** можно включить в Файл > Включить автосохранение
- Настройте интервал автосохранения в Файл > Настройки автосохранения

### Открытие проектов

- **Ctrl+O** открывает папку проекта
- Последние проекты перечислены в **Файл > Последние проекты**
- ZIP-проекты можно открыть через **Файл > Открыть Zip-проект**

## 5. Спрайты

Спрайты — это визуальные изображения, используемые игровыми объектами. Они могут быть статическими изображениями или анимационными полосами с несколькими кадрами.

### Создание спрайта

1. Щёлкните правой кнопкой по **Спрайты** в дереве ресурсов и выберите **Импорт спрайта...**
2. Выберите файл изображения (PNG, JPG, BMP или GIF)
3. Спрайт появится в дереве ресурсов и откроется в редакторе спрайтов

### Редактор спрайтов

Редактор спрайтов имеет четыре основные области:

- **Панель инструментов (слева):** Инструменты рисования
- **Холст (по центру):** Область редактирования с масштабированием и сеткой
- **Палитра цветов (справа):** Основной/фоновый цвета и образцы
- **Временная шкала кадров (внизу):** Управление кадрами анимации

#### Инструменты рисования

| Инструмент | Сочетание | Описание |
|------------|-----------|----------|
| Карандаш | P | Стандартное рисование пикселей |
| Ластик | E | Удаление пикселей (делает прозрачными) |
| Пипетка | I | Выбор цвета с холста |
| Заливка | G | Заливка связанных областей |
| Линия | L | Рисование прямых линий |
| Прямоугольник | R | Рисование прямоугольников (контур или заливка) |
| Эллипс | O | Рисование эллипсов (контур или заливка) |
| Выделение | S | Выделение, вырезание, копирование, вставка областей |

#### Размер кисти

Размер кисти варьируется от 1 до 16 пикселей и влияет на инструменты «Карандаш», «Ластик» и «Линия».

#### Режим заливки

Переключение между контурным и заливочным режимом для инструментов «Прямоугольник» и «Эллипс» с помощью кнопки **Заливка**.

#### Палитра цветов

- **Левый щелчок** по образцу основного цвета для выбора цвета рисования
- **Правый щелчок** по образцу фонового цвета для выбора вторичного цвета
- **Кнопка X** меняет местами основной и фоновый цвета
- Щёлкните любой цвет в быстрой палитре, чтобы выбрать его
- Дважды щёлкните по образцу палитры, чтобы настроить его
- Полный выбор цвета RGBA с поддержкой альфа-канала (прозрачности)

### Кадры анимации

Спрайты могут иметь несколько кадров анимации, расположенных в горизонтальную полосу.

**Элементы управления временной шкалой кадров:**
- **+ (Добавить):** Добавить новый пустой кадр
- **D (Дублировать):** Копировать текущий кадр
- **- (Удалить):** Удалить текущий кадр (минимум 1 кадр)
- **Воспроизведение/Стоп:** Предварительный просмотр анимации
- Счётчик кадров показывает текущий/общее число кадров

**Свойства анимации:**
- **Количество кадров:** Число кадров анимации
- **Скорость анимации:** Кадров в секунду (по умолчанию: 10 FPS)
- **Тип анимации:** single, strip_h (горизонтальная), strip_v (вертикальная) или grid

**Поддержка анимированных GIF:** Импортируйте анимированные GIF-файлы напрямую. Все кадры извлекаются автоматически с обработкой прозрачности.

### Начало координат спрайта

Начало координат — это точка привязки, используемая для размещения и вращения в игре.

**Предустановленные позиции:**
- Верхний левый (0, 0)
- Верхний центр
- Центр (по умолчанию для большинства спрайтов)
- Центр-низ
- Нижний левый
- Нижний правый
- Пользовательский (ручной ввод X/Y)

Начало координат отображается как перекрестие на холсте.

### Управление холстом

- **Ctrl+Колесо мыши:** Масштабирование (от 1x до 64x)
- **Переключатель сетки:** Показать/скрыть пиксельную сетку (видна при масштабе 4x и выше)
- **Отражение Г/В:** Отразить текущий кадр горизонтально или вертикально
- **Изменение размера/Масштаб:** Изменение размеров спрайта с настройками масштаба или размера холста
- **Экспорт PNG…:** Экспортировать текущее изображение как PNG-файл (также доступно через правый клик на холсте)

### Свойства спрайта (сохраняемые)

| Свойство | Описание |
|----------|----------|
| name | Имя ресурса |
| file_path | Путь к файлу PNG-полосы |
| width | Полная ширина полосы |
| height | Высота кадра |
| frame_width | Ширина одного кадра |
| frame_height | Высота одного кадра |
| frames | Количество кадров |
| animation_type | single, strip_h, strip_v, grid |
| speed | FPS анимации |
| origin_x | Координата X начала координат |
| origin_y | Координата Y начала координат |

## 6. Звуки

Звуки — это аудиофайлы, используемые для звуковых эффектов и фоновой музыки.

### Импорт звуков

1. Щёлкните правой кнопкой по **Звуки** в дереве ресурсов и выберите **Импорт звука...**
2. Выберите аудиофайл (WAV, OGG, MP3)
3. Звук добавляется в проект

### Свойства звука

| Свойство | Описание |
|----------|----------|
| name | Имя звукового ресурса |
| file_path | Путь к аудиофайлу |
| kind | "sound" (эффект) или "music" (потоковое воспроизведение) |
| volume | Громкость по умолчанию (от 0.0 до 1.0) |

**Звуковые эффекты** загружаются в память для мгновенного воспроизведения. **Музыкальные** файлы воспроизводятся потоком с диска, и одновременно может играть только один.

## 7. Фоны

Фоны — это изображения, используемые за игровыми объектами. Они также могут служить наборами тайлов для уровней на основе тайлов.

### Импорт фонов

1. Щёлкните правой кнопкой по **Фоны** в дереве ресурсов и выберите **Импорт фона...**
2. Выберите файл изображения

### Настройка набора тайлов

Фоны могут быть настроены как наборы тайлов со следующими свойствами:

| Свойство | Описание |
|----------|----------|
| tile_width | Ширина каждого тайла (по умолчанию: 16) |
| tile_height | Высота каждого тайла (по умолчанию: 16) |
| h_offset | Горизонтальное смещение до первого тайла |
| v_offset | Вертикальное смещение до первого тайла |
| h_sep | Горизонтальный промежуток между тайлами |
| v_sep | Вертикальный промежуток между тайлами |
| use_as_tileset | Включить режим набора тайлов |

## 8. Объекты

Объекты определяют игровые сущности со свойствами, событиями и действиями. Каждый объект может иметь спрайт для визуального представления и содержит обработчики событий, определяющие его поведение.

### Создание объекта

1. Щёлкните правой кнопкой по **Объекты** в дереве ресурсов и выберите **Создать объект...**
2. Введите имя объекта
3. Объект откроется в редакторе объектов

### Свойства объекта

| Свойство | По умолчанию | Описание |
|----------|--------------|----------|
| Спрайт | Нет | Визуальный спрайт для отображения |
| Видимый | Да | Отображаются ли экземпляры |
| Твёрдый | Нет | Блокирует ли объект движение |
| Постоянный | Нет | Сохраняются ли экземпляры при смене комнат |
| Глубина | 0 | Порядок отрисовки (больше = позади, меньше = впереди) |
| Родитель | Нет | Родительский объект для наследования |

### Родительские объекты

Объекты могут наследовать от родительского объекта. Дочерние объекты получают все события столкновений родителя. Это полезно для создания иерархий, например:

```
obj_enemy (parent - has collision with obj_player)
  ├── obj_enemy_melee (inherits collision handling)
  └── obj_enemy_ranged (inherits collision handling)
```

### Добавление событий

1. Откройте редактор объектов
2. Нажмите **Добавить событие** на панели событий
3. Выберите тип события из списка (см. [Справочник событий](#10-справочник-событий))
4. Событие появится в дереве событий

### Добавление действий к событиям

1. Выберите событие в дереве событий
2. Нажмите **Добавить действие** или щёлкните правой кнопкой и выберите **Добавить действие**
3. Выберите тип действия из категоризированного списка
4. Настройте параметры действия в диалоговом окне
5. Нажмите ОК, чтобы добавить действие

Действия выполняются по порядку сверху вниз при срабатывании события.

### Условная логика

Действия поддерживают условный поток if/else:

1. Добавьте условное действие (например, **Если столкновение в**, **Проверить переменную**)
2. Добавьте действие **Начало блока** (открывающая скобка)
3. Добавьте действия, которые выполняются при истинном условии
4. Добавьте действие **Иначе** (необязательно)
5. Добавьте **Начало блока** для ветки «иначе»
6. Добавьте действия для случая ложного условия
7. Добавьте действия **Конец блока** для закрытия каждого блока

Пример последовательности действий:
```
If Collision At (self.x, self.y + 1, "solid")
  Start Block
    Set Vertical Speed (0)
  End Block
Else
  Start Block
    Set Vertical Speed (vspeed + 0.5)
  End Block
```

### Просмотр кода

Установите флажок **Просмотр кода** в редакторе объектов, чтобы увидеть сгенерированный код Python для всех событий и действий. Это полезно для понимания того, как визуальные действия преобразуются в код.

| Порт X/Y | Позиция на экране для этого вида |
| Порт Ш/В | Размер на экране для этого вида |
| Следить за объектом | Объект, за которым следит камера |
| Граница Г/В | Поля прокрутки вокруг отслеживаемого объекта |
| Скорость Г/В | Максимальная скорость прокрутки камеры (-1 = мгновенно) |

| Область | Синтаксис | Описание |
|---------|-----------|----------|
| Self | `self.variable` или просто `variable` | Переменная текущего экземпляра |
| Other | `other.variable` | Другой экземпляр при столкновении |
| Global | `global.variable` | Глобальная переменная игры |

Встроенные переменные экземпляра: `x`, `y`, `hspeed`, `vspeed`, `direction`, `speed`, `gravity`, `friction`, `visible`, `depth`, `image_index`, `image_speed`, `scale_x`, `scale_y`

## 12. Тестирование и запуск игр

### Быстрый тест (F5)

Нажмите **F5** или кнопку **Тест**, чтобы мгновенно запустить игру. Откроется отдельное окно Pygame с вашей игрой.

- Нажмите **Escape**, чтобы остановить игру и вернуться в IDE
- Строка состояния IDE показывает «Игра запущена...», пока игра активна

### Режим отладки (F6)

Нажмите **F6** для режима отладки, который показывает дополнительный вывод в консоли, включая:
- Журнал выполнения событий
- Детали обнаружения столкновений
- Значения параметров действий
- Создание и уничтожение экземпляров

### Порядок выполнения игры

Каждый кадр следует порядку событий GameMaker 7.0:

1. События **Begin Step**
2. Обратный отсчёт **Alarm** и их срабатывание
3. События **Step**
4. События ввода **Keyboard/Mouse**
5. **Движение** (физика: гравитация, трение, hspeed/vspeed)
6. Обнаружение **столкновений** и события
7. События **End Step**
8. События **Destroy** для отмеченных экземпляров
9. События **Draw** и отрисовка

### Заголовок окна

Заголовок игрового окна может отображать значения счёта, жизней и здоровья. Включите это в настройках отображения заголовка в параметрах проекта или с помощью действий счёта/жизней/здоровья.

## 13. Экспорт игр

### Экспорт в HTML5

**Файл > Экспорт в HTML5**

Создаёт единый автономный HTML-файл, который работает в любом веб-браузере.

- Все спрайты встроены как данные base64
- Данные игры сжаты с помощью gzip
- Движок JavaScript обрабатывает рендеринг через HTML5 Canvas
- Сервер не требуется — просто откройте файл в браузере

### Экспорт в EXE (Windows)

**Файл > Экспорт проекта** или **Сборка > Экспорт игры**

Создаёт автономный исполняемый файл Windows с помощью PyInstaller.

**Требования:** Должны быть установлены PyInstaller и Kivy.

**Процесс:**
1. Генерирует игру на основе Kivy из вашего проекта
2. Упаковывает среду выполнения Python и все зависимости
3. Создаёт единый EXE-файл (может занять 5-10 минут)

**Параметры:**
- Консоль отладки (показывает окно терминала для отладки)
- Пользовательская иконка
- Сжатие UPX (уменьшает размер файла)

### Мобильный экспорт (Kivy)

**Файл > Экспорт в Kivy**

Генерирует проект Kivy для мобильного развёртывания.

**Результат включает:**
- Код игры на Python, адаптированный для Kivy
- Пакет ресурсов, оптимизированный для мобильных устройств
- Конфигурация `buildozer.spec` для сборки под Android/iOS

**Для сборки под Android:**
```bash
cd exported_project
buildozer android debug
```

### Экспорт в ZIP

**Файл > Экспорт в Zip**

Упаковывает весь проект как ZIP-архив для обмена или резервного копирования. ZIP можно повторно открыть через **Файл > Открыть Zip-проект**.

### Экспорт в Aseba (робот Thymio)

Для проектов Thymio экспортирует файлы кода AESL, совместимые с Aseba Studio.

## 15. Поддержка робота Thymio

PyGameMaker включает поддержку образовательного робота Thymio, позволяя симулировать и программировать роботов Thymio в IDE.

### Что такое Thymio?

Thymio — это небольшой образовательный робот с датчиками, светодиодами, моторами и кнопками. PyGameMaker может симулировать поведение Thymio и экспортировать код для запуска на реальных роботах.

### Включение Thymio

Перейдите в **Инструменты > Программирование Thymio > Показать вкладку Thymio в редакторе объектов**, чтобы включить функции Thymio в редакторе объектов.

### Симулятор Thymio

Симулятор моделирует физического робота Thymio:

**Характеристики:**
- Размер робота: 110x110 пикселей (11 см x 11 см)
- База колёс: 95 пикселей
- Диапазон скорости мотора: от -500 до +500

### Датчики Thymio

| Датчик | Количество | Диапазон | Описание |
|--------|------------|----------|----------|
| Proximity | 7 | 0-4000 | Горизонтальные датчики расстояния (дальность 10 см) |
| Ground | 2 | 0-1023 | Определяет светлые/тёмные поверхности |
| Buttons | 5 | 0/1 | Вперёд, назад, влево, вправо, центр |

### События Thymio

| Событие | Срабатывает, когда |
|---------|-------------------|
| Button Forward/Backward/Left/Right/Center | Нажата ёмкостная кнопка |
| Any Button | Изменяется состояние любой кнопки |
| Proximity Update | Обновляются датчики расстояния (10 Гц) |
| Ground Update | Обновляются датчики поверхности (10 Гц) |
| Tap | Акселерометр обнаруживает удар/толчок |
| Sound Detected | Микрофон обнаруживает звук |
| Timer 0/1 | Истекает период таймера |
| Sound Finished | Завершается воспроизведение звука |
| Message Received | Получена IR-коммуникация |

### Действия Thymio

**Управление моторами:**
- Установить скорость мотора (левый, правый — независимо)
- Двигаться вперёд / назад
- Повернуть влево / вправо
- Остановить моторы

**Управление светодиодами:**
- Установить верхний светодиод (цвет RGB)
- Установить нижний левый/правый светодиод
- Установить круговые светодиоды (8 светодиодов по периметру)
- Выключить все светодиоды

**Звук:**
- Воспроизвести тон (частота, длительность)
- Воспроизвести системный звук
- Остановить звук

**Условия датчиков:**
- Если расстояние (датчик, порог, сравнение)
- Если поверхность тёмная / светлая
- Если кнопка нажата / отпущена

**Таймер:**
- Установить период таймера (таймер 0 или 1, период в мс)

### Экспорт на реального Thymio

1. Экспортируйте проект Thymio через экспортёр Aseba
2. Откройте сгенерированный файл `.aesl` в Aseba Studio
3. Подключите Thymio через USB
4. Нажмите **Load**, затем **Run**

## 19. Устранение неполадок

### Игра не запускается (F5)

- Убедитесь, что в проекте есть хотя бы одна комната с экземплярами
- Проверьте, что объектам назначены спрайты
- Просмотрите вывод консоли (режим отладки F6) для сообщений об ошибках

### Спрайты не отображаются

- Убедитесь, что файл спрайта существует в папке `sprites/`
- Проверьте, что объекту назначен спрайт в его свойствах
- Убедитесь, что экземпляр имеет `visible = true`

### Столкновения не работают

- Убедитесь, что целевой объект отмечен как **Твёрдый**, если используется столкновение на основе твёрдых тел
- Проверьте, что у вас настроено событие **Столкновение с** для правильного объекта
- Убедитесь, что экземпляры действительно перекрываются (используйте режим отладки)

### Звук не воспроизводится

- Убедитесь, что звуковой файл существует и имеет поддерживаемый формат (WAV, OGG, MP3)
- Проверьте, что pygame.mixer успешно инициализирован (см. вывод консоли)
- Музыкальные файлы воспроизводятся потоком с диска — убедитесь, что путь к файлу правильный

### Ошибка экспорта

- **Экспорт в EXE:** Убедитесь, что установлен PyInstaller (`pip install pyinstaller`)
- **Экспорт в Kivy:** Убедитесь, что установлен Kivy (`pip install kivy`)
- **Экспорт в HTML5:** Проверьте консоль на наличие ошибок кодировки
- Все виды экспорта требуют корректного проекта с хотя бы одной комнатой

### Проблемы производительности

- Уменьшите количество экземпляров в комнатах
- Используйте систему столкновений с пространственной сеткой (включена по умолчанию)
- Избегайте ресурсоёмких операций в событиях Step (выполняются 60 раз в секунду)
- Используйте будильники для периодических задач вместо подсчёта кадров в Step
