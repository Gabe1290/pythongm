## 1. Вступ

PyGameMaker — це навчальне середовище розробки ігор, натхненне GameMaker. Воно дозволяє створювати 2D-ігри візуально за допомогою системи подій та дій методом перетягування, без необхідності писати код. Воно розроблене з двома цілями:

- **Навчання розробці ігор** візуально через інтуїтивний інтерфейс
- **Навчання програмуванню на Python** через відкритий вихідний код IDE

PyGameMaker використовує PySide6 (Qt) для інтерфейсу IDE та Pygame для ігрового середовища виконання. Ігри можна експортувати як окремі виконувані файли, мобільні додатки або веб-ігри HTML5.

### Основні можливості

- Візуальне програмування подій/дій (80+ вбудованих дій)
- Редактор спрайтів з підтримкою анімації
- Редактор кімнат з розміщенням екземплярів, шарами тайлів та прокруткою фону
- Попередній перегляд гри в реальному часі клавішею F5
- Експорт у Windows EXE, мобільні платформи (Android/iOS через Kivy), HTML5 та роботи Thymio
- Інтеграція з Google Blockly для візуальних блоків коду
- Симулятор навчального робота Thymio
- Багатомовний інтерфейс (8+ мов)
- Темна та світла теми

## 2. Встановлення та налаштування

### Вимоги

- Python 3.10 або вище
- PySide6
- Pygame
- Pillow (PIL)

### Встановлення залежностей

```bash
pip install PySide6 pygame Pillow
```

### Додаткові залежності

Для функцій експорту:

```bash
pip install pyinstaller    # For EXE export
pip install kivy           # For mobile export
pip install buildozer      # For Android builds
pip install jinja2         # For code generation templates
```

### Запуск IDE

```bash
python main.py
```

Вікно IDE відкривається з розмірами за замовчуванням 1400x900 пікселів. Позиція та розмір вікна зберігаються між сеансами.

## 3. Огляд IDE

### Компонування вікна

IDE використовує тріпанельне компонування:

```
+-------------------+---------------------------+------------------+
|   Дерево ресурсів |      Область редагування  |   Властивості    |
|   (Ліва панель)   |      (Центральна панель)  |   (Права панель) |
|                   |                           |                  |
|   - Спрайти       |   Редактори з вкладками   |   Контекстно-    |
|   - Звуки         |   для спрайтів, об'єктів  |   залежні         |
|   - Фони          |   та кімнат               |   властивості     |
|   - Об'єкти       |                           |                  |
|   - Кімнати       |                           |                  |
|   - Скрипти       |                           |                  |
|   - Шрифти        |                           |                  |
+-------------------+---------------------------+------------------+
|                       Рядок стану                                |
+------------------------------------------------------------------+
```

- **Ліва панель (Дерево ресурсів):** Відображає всі ресурси проєкту, організовані за типом. Двічі клацніть ресурс, щоб відкрити його в редакторі.
- **Центральна панель (Область редагування):** Робочий простір із вкладками, де ви редагуєте спрайти, об'єкти та кімнати. За замовчуванням показується вкладка привітання.
- **Права панель (Властивості):** Показує контекстно-залежні властивості для поточного активного редактора. Може бути згорнута.
- **Рядок стану:** Показує стан поточної операції та назву проєкту.

## 4. Робота з проєктами

### Створення нового проєкту

1. Оберіть **Файл > Новий проєкт** (Ctrl+N)
2. Введіть **Назву проєкту**
3. Оберіть **Розташування** для папки проєкту
4. Оберіть **Шаблон** (необов'язково):
   - **Порожній проєкт** — Чистий проєкт
   - **Шаблон платформера** — Попередньо налаштований для платформерів
   - **Шаблон гри зверху** — Попередньо налаштований для ігор з видом зверху
5. Натисніть **Створити проєкт**

### Структура проєкту

```
MyProject/
├── project.json          # Main project file
├── sprites/              # Sprite images and metadata
│   ├── player.png
│   └── player.json
├── objects/              # Object definitions
│   └── obj_player.json
├── rooms/                # Room layout data
│   └── room_start.json
├── sounds/               # Audio files and metadata
├── backgrounds/          # Background images
└── thumbnails/           # Auto-generated previews
```

### Збереження проєктів

- **Ctrl+S** зберігає проєкт у поточному розташуванні
- **Ctrl+Shift+S** зберігає в нове розташування
- **Автозбереження** можна увімкнути в Файл > Увімкнути автозбереження
- Налаштуйте інтервал автозбереження в Файл > Налаштування автозбереження

### Відкриття проєктів

- **Ctrl+O** відкриває папку проєкту
- Останні проєкти перелічені в **Файл > Останні проєкти**
- ZIP-проєкти можна відкрити через **Файл > Відкрити Zip-проєкт**

## 5. Спрайти

Спрайти — це візуальні зображення, які використовуються ігровими об'єктами. Вони можуть бути статичними зображеннями або анімаційними стрічками з кількома кадрами.

### Створення спрайту

1. Клацніть правою кнопкою миші на **Спрайти** в дереві ресурсів та оберіть **Імпорт спрайту...**
2. Оберіть файл зображення (PNG, JPG, BMP або GIF)
3. Спрайт з'являється в дереві ресурсів та відкривається в редакторі спрайтів

### Редактор спрайтів

Редактор спрайтів має чотири основні області:

- **Панель інструментів (зліва):** Інструменти малювання
- **Полотно (центр):** Область редагування зі збільшенням та сіткою
- **Палітра кольорів (справа):** Кольори переднього/заднього плану та зразки
- **Шкала часу кадрів (знизу):** Управління кадрами анімації

#### Інструменти малювання

| Інструмент | Комбінація клавіш | Опис |
|------|----------|-------------|
| Олівець | P | Стандартне піксельне малювання |
| Гумка | E | Видалення пікселів (зробити прозорими) |
| Піпетка | I | Вибір кольору з полотна |
| Заливка | G | Заповнення з'єднаних областей |
| Лінія | L | Малювання прямих ліній |
| Прямокутник | R | Малювання прямокутників (контур або заливка) |
| Еліпс | O | Малювання еліпсів (контур або заливка) |
| Виділення | S | Виділення, вирізання, копіювання, вставка областей |

#### Розмір пензля

Розмір пензля від 1 до 16 пікселів, впливає на інструменти Олівець, Гумка та Лінія.

#### Режим заливки

Перемикання між контуром та режимом заливки для інструментів Прямокутник та Еліпс за допомогою кнопки **Заливка**.

#### Палітра кольорів

- **Ліва кнопка миші** на зразку переднього плану для вибору кольору малювання
- **Права кнопка миші** на зразку фону для вибору додаткового кольору
- **Кнопка X** міняє місцями кольори переднього та заднього плану
- Клацніть будь-який колір у швидкій палітрі, щоб обрати його
- Двічі клацніть зразок палітри, щоб налаштувати його
- Повний вибір кольору RGBA з підтримкою альфа-каналу (прозорості)

### Кадри анімації

Спрайти можуть мати кілька кадрів анімації, розташованих у горизонтальну стрічку.

**Елементи управління шкалою часу кадрів:**
- **+ (Додати):** Додати новий порожній кадр
- **D (Дублювати):** Копіювати поточний кадр
- **- (Видалити):** Видалити поточний кадр (мінімум 1 кадр)
- **Відтворити/Зупинити:** Попередній перегляд анімації
- Лічильник кадрів показує поточний/загальну кількість кадрів

**Властивості анімації:**
- **Кількість кадрів:** Кількість кадрів анімації
- **Швидкість анімації:** Кадрів на секунду (за замовчуванням: 10 FPS)
- **Тип анімації:** single, strip_h (горизонтальна), strip_v (вертикальна) або grid

**Підтримка анімованих GIF:** Імпортуйте анімовані файли GIF безпосередньо. Усі кадри витягуються автоматично з обробкою прозорості.

### Початок координат спрайту

Точка початку координат — це якірна позиція, яка використовується для розміщення та обертання в грі.

**Попередньо задані позиції:**
- Верхній лівий кут (0, 0)
- Верхній центр
- Центр (за замовчуванням для більшості спрайтів)
- Центр-низ
- Нижній лівий кут
- Нижній правий кут
- Користувацький (ручне введення X/Y)

Початок координат відображається як перехрестя на полотні.

### Управління полотном

- **Ctrl+Колесо миші:** Збільшення/зменшення (від 1x до 64x)
- **Перемикач сітки:** Показати/сховати піксельну сітку (видима при збільшенні 4x і вище)
- **Дзеркало Г/В:** Відобразити поточний кадр горизонтально або вертикально
- **Зміна розміру/Масштаб:** Зміна розмірів спрайту з параметрами масштабування або зміни розміру полотна

### Властивості спрайту (збережені)

| Властивість | Опис |
|----------|-------------|
| name | Назва ресурсу |
| file_path | Шлях до файлу PNG-стрічки |
| width | Повна ширина стрічки |
| height | Висота кадру |
| frame_width | Ширина одного кадру |
| frame_height | Висота одного кадру |
| frames | Кількість кадрів |
| animation_type | single, strip_h, strip_v, grid |
| speed | Кадрів на секунду анімації |

| origin_y | Координата Y початку координат |

## 8. Об'єкти

Об'єкти визначають ігрові сутності з властивостями, подіями та діями. Кожен об'єкт може мати спрайт для візуального представлення та містить обробники подій, що визначають його поведінку.

### Створення об'єкта

1. Клацніть правою кнопкою миші на **Об'єкти** в дереві ресурсів та оберіть **Створити об'єкт...**
2. Введіть назву для об'єкта
3. Об'єкт відкривається в редакторі об'єктів

### Властивості об'єкта

| Властивість | За замовчуванням | Опис |
|----------|---------|-------------|
| Спрайт | Немає | Візуальний спрайт для відображення |
| Видимий | Так | Чи малюються екземпляри |
| Твердий | Ні | Чи блокує об'єкт рух |
| Постійний | Ні | Чи зберігаються екземпляри при зміні кімнати |
| Глибина | 0 | Порядок малювання (більше = позаду, менше = попереду) |
| Батько | Немає | Батьківський об'єкт для наслідування |

### Батьківські об'єкти

Об'єкти можуть наслідувати від батьківського об'єкта. Дочірні об'єкти отримують усі події зіткнень батька. Це корисно для створення ієрархій, таких як:

```
obj_enemy (parent - has collision with obj_player)
  ├── obj_enemy_melee (inherits collision handling)
  └── obj_enemy_ranged (inherits collision handling)
```

### Додавання подій

1. Відкрийте редактор об'єктів
2. Натисніть **Додати подію** на панелі подій
3. Оберіть тип події зі списку (див. [Довідник подій](#10-довідник-подій))
4. Подія з'являється в дереві подій

### Додавання дій до подій

1. Оберіть подію в дереві подій
2. Натисніть **Додати дію** або клацніть правою кнопкою миші та оберіть **Додати дію**
3. Оберіть тип дії з категоризованого списку
4. Налаштуйте параметри дії в діалоговому вікні
5. Натисніть OK, щоб додати дію

Дії виконуються послідовно зверху вниз, коли спрацьовує подія.

### Умовна логіка

Дії підтримують умовний потік виконання if/else:

1. Додайте умовну дію (наприклад, **Якщо зіткнення в**, **Перевірити змінну**)
2. Додайте дію **Початок блоку** (відкриваюча дужка)
3. Додайте дії, що виконуються при істинній умові
4. Додайте дію **Інакше** (необов'язково)
5. Додайте **Початок блоку** для гілки else
6. Додайте дії для хибного випадку
7. Додайте дії **Кінець блоку**, щоб закрити кожен блок

Приклад послідовності дій:
```
If Collision At (self.x, self.y + 1, "solid")
  Start Block
    Set Vertical Speed (0)
  End Block
Else
  Start Block
    Set Vertical Speed (vspeed + 0.5)
  End Block
```

### Перегляд коду

Увімкніть опцію **Перегляд коду** в редакторі об'єктів, щоб побачити згенерований код Python для всіх подій та дій. Це корисно для розуміння того, як візуальні дії перетворюються на код.

## 9. Кімнати

Кімнати — це сцени або рівні вашої гри. Кожна кімната має фон, розміщені екземпляри об'єктів та необов'язкові шари тайлів.

### Створення кімнати

1. Клацніть правою кнопкою миші на **Кімнати** в дереві ресурсів та оберіть **Створити кімнату...**
2. Введіть назву для кімнати
3. Кімната відкривається в редакторі кімнат

### Властивості кімнати

| Властивість | За замовчуванням | Опис |
|----------|---------|-------------|
| Ширина | 1024 | Ширина кімнати в пікселях |
| Висота | 768 | Висота кімнати в пікселях |
| Колір фону | #87CEEB (блакитний) | Колір заливки позаду всього |
| Фонове зображення | Немає | Необов'язкове фонове зображення |
| Постійна | Ні | Зберігати стан при виході |

### Розміщення екземплярів

1. Натисніть на об'єкт у **Палітрі об'єктів** (ліва частина редактора кімнат)
2. Натисніть на полотно кімнати, щоб розмістити екземпляр
3. Продовжуйте натискати, щоб розмістити більше копій
4. Виділіть розміщені екземпляри, щоб перемістити або налаштувати їх

### Властивості екземпляра

При виділенні розміщеного екземпляра:

| Властивість | Діапазон | Опис |
|----------|-------|-------------|
| Позиція X | 0-9999 | Горизонтальна позиція |
| Позиція Y | 0-9999 | Вертикальна позиція |
| Видимий | Так/Ні | Видимість екземпляра |
| Обертання | 0-360 | Обертання в градусах |
| Масштаб X | 10%-1000% | Горизонтальний масштаб |
| Масштаб Y | 10%-1000% | Вертикальний масштаб |

### Сітка та прив'язка

- **Перемикач сітки:** Натисніть кнопку сітки, щоб показати/сховати сітку розміщення
- **Перемикач прив'язки:** Натисніть кнопку прив'язки, щоб увімкнути/вимкнути прив'язку до сітки
- **Розмір сітки:** 32x32 пікселі за замовчуванням (налаштовується в Параметрах)

### Операції з екземплярами

| Дія | Комбінація клавіш | Опис |
|--------|----------|-------------|
| Перемістити | Перетягнути | Перемістити екземпляр на нову позицію |
| Множинне виділення | Shift+Клік | Додати/видалити з виділення |
| Виділення рамкою | Перетягнути порожню область | Виділити всі екземпляри в прямокутнику |
| Видалити | Delete | Видалити вибрані екземпляри |
| Вирізати | Ctrl+X | Вирізати в буфер обміну |
| Копіювати | Ctrl+C | Копіювати в буфер обміну |
| Вставити | Ctrl+V | Вставити з буфера обміну |
| Дублювати | Ctrl+D | Дублювати вибрані екземпляри |
| Очистити все | Кнопка панелі інструментів | Видалити всі екземпляри (з підтвердженням) |

### Шари фону

Кімнати підтримують до 8 шарів фону (індекси 0-7), кожен з незалежними налаштуваннями:

| Властивість | Опис |
|----------|-------------|
| Фонове зображення | Який ресурс фону використовувати |
| Видимий | Показати/сховати шар |
| Передній план | Якщо так, малюється попереду екземплярів |
| Мозаїка по горизонталі | Повторювати по ширині кімнати |
| Мозаїка по вертикалі | Повторювати по висоті кімнати |
| Швидкість горизонтальної прокрутки | Горизонтальна прокрутка пікселів за кадр |
| Швидкість вертикальної прокрутки | Вертикальна прокрутка пікселів за кадр |
| Розтягнути | Масштабувати на всю кімнату |
| Зсув X / Y | Зсув позиції шару |

### Шари тайлів

Для рівнів на основі тайлів:

1. Натисніть **Палітра тайлів...**, щоб відкрити вибір тайлів
2. Оберіть набір тайлів (фон, позначений як набір тайлів)
3. Встановіть ширину та висоту тайлів
4. Натисніть на тайл у палітрі, щоб обрати його
5. Натисніть на кімнату, щоб розмістити тайли
6. Оберіть **Шар** (0-7) для тайлів

### Порядок кімнат

Кімнати виконуються в порядку, в якому вони з'являються в дереві ресурсів. Перша кімната є стартовою.

- Клацніть правою кнопкою миші на кімнаті та використовуйте **Вгору/Вниз/На початок/В кінець** для зміни порядку
- Використовуйте дії **Наступна кімната** та **Попередня кімната** для навігації між кімнатами під час виконання

### Система видів

Кімнати підтримують до 8 видів камери (як у GameMaker):

| Властивість | Опис |
|----------|-------------|
| Видимий | Увімкнути/вимкнути цей вид |
| Вид X/Y | Позиція камери в кімнаті |
| Вид Ш/В | Розмір області огляду камери |
| Порт X/Y | Позиція на екрані для цього виду |
| Порт Ш/В | Розмір на екрані для цього виду |
| Об'єкт стеження | Об'єкт для відстеження камерою |
| Горизонтальна/Вертикальна межа | Відступ прокрутки навколо об'єкта стеження |
| Горизонтальна/Вертикальна швидкість | Максимальна швидкість прокрутки камери (-1 = миттєво) |

## 12. Тестування та запуск ігор

### Швидкий тест (F5)

Натисніть **F5** або кнопку **Тест**, щоб миттєво запустити вашу гру. Відкриється окреме вікно Pygame з вашою грою.

- Натисніть **Escape**, щоб зупинити гру та повернутися до IDE
- Рядок стану IDE показує «Гра запущена...» поки гра активна

### Режим налагодження (F6)

Натисніть **F6** для режиму налагодження, який показує додатковий вивід у консоль, включаючи:
- Журнал виконання подій
- Деталі виявлення зіткнень
- Значення параметрів дій
- Створення та знищення екземплярів

### Порядок виконання гри

Кожен кадр відповідає порядку подій GameMaker 7.0:

1. Події **Begin Step**
2. Зворотний відлік та спрацювання **Alarm**
3. Події **Step**
4. Події введення **Keyboard/Mouse**
5. **Рух** (фізика: гравітація, тертя, hspeed/vspeed)
6. Виявлення та події **зіткнень**
7. Події **End Step**
8. Події **Destroy** для позначених екземплярів
9. Події **Draw** та рендеринг

### Заголовок вікна

Заголовок ігрового вікна може показувати значення рахунку, життів та здоров'я. Увімкніть це в налаштуваннях відображення заголовка в налаштуваннях проєкту або використовуючи дії рахунку/життів/здоров'я.

## 13. Експорт ігор

### Експорт HTML5

**Файл > Експорт як HTML5**

Створює один самодостатній HTML-файл, який працює в будь-якому веб-браузері.

- Усі спрайти вбудовані як дані base64
- Ігрові дані стиснуті за допомогою gzip
- Рушій JavaScript обробляє рендеринг через HTML5 Canvas
- Сервер не потрібен — просто відкрийте файл у браузері

### Експорт EXE (Windows)

**Файл > Експорт проєкту** або **Збірка > Експортувати гру**

Створює окремий виконуваний файл Windows за допомогою PyInstaller.

**Вимоги:** Мають бути встановлені PyInstaller та Kivy.

**Процес:**
1. Генерує гру на базі Kivy з вашого проєкту
2. Пакує середовище виконання Python та всі залежності
3. Створює один EXE-файл (може зайняти 5-10 хвилин)

**Параметри:**
- Консоль налагодження (показує вікно терміналу для налагодження)
- Користувацька іконка
- Стиснення UPX (зменшує розмір файлу)

### Мобільний експорт (Kivy)

**Файл > Експорт у Kivy**

Генерує проєкт Kivy для мобільного розгортання.

**Вивід включає:**
- Ігровий код Python, адаптований для Kivy
- Пакет ресурсів, оптимізований для мобільних пристроїв
- Конфігурацію `buildozer.spec` для збірок Android/iOS

**Для збірки під Android:**
```bash
cd exported_project
buildozer android debug
```

### Експорт ZIP

**Файл > Експорт як Zip**

Пакує весь проєкт як ZIP-архів для поширення або резервного копіювання. ZIP можна повторно відкрити через **Файл > Відкрити Zip-проєкт**.

### Експорт Aseba (робот Thymio)

Для проєктів робота Thymio експортує файли коду AESL, сумісні з Aseba Studio.

## 15. Підтримка робота Thymio

PyGameMaker включає підтримку навчального робота Thymio, що дозволяє симулювати та програмувати роботів Thymio в IDE.

### Що таке Thymio?

Thymio — це невеликий навчальний робот з датчиками, світлодіодами, моторами та кнопками. PyGameMaker може симулювати поведінку Thymio та експортувати код для запуску на реальних роботах.

### Увімкнення Thymio

Перейдіть до **Інструменти > Програмування Thymio > Показати вкладку Thymio в редакторі об'єктів**, щоб увімкнути функції Thymio в редакторі об'єктів.

### Симулятор Thymio

Симулятор моделює фізичного робота Thymio:

**Специфікації:**
- Розмір робота: 110x110 пікселів (11 см x 11 см)
- Колісна база: 95 пікселів
- Діапазон швидкості мотора: від -500 до +500

### Датчики Thymio

| Датчик | Кількість | Діапазон | Опис |
|--------|-------|-------|-------------|
| Наближення | 7 | 0-4000 | Горизонтальні датчики відстані (діапазон 10 см) |
| Наземний | 2 | 0-1023 | Виявлення світлих/темних поверхонь |
| Кнопки | 5 | 0/1 | Вперед, назад, ліворуч, праворуч, центр |

### Події Thymio

| Подія | Спрацьовує коли |
|-------|---------------|
| Button Forward/Backward/Left/Right/Center | Ємнісна кнопка натиснута |
| Any Button | Будь-який стан кнопки змінився |
| Proximity Update | Датчики наближення оновлюються (10 Гц) |
| Ground Update | Наземні датчики оновлюються (10 Гц) |
| Tap | Акселерометр виявляє удар/поштовх |
| Sound Detected | Мікрофон виявляє звук |
| Timer 0/1 | Період таймера закінчився |
| Sound Finished | Відтворення звуку завершено |
| Message Received | Отримано ІЧ-повідомлення |

### Дії Thymio

**Управління моторами:**
- Встановити швидкість мотора (лівий, правий незалежно)
- Рухатися вперед / назад
- Повернути ліворуч / праворуч
- Зупинити мотори

**Управління світлодіодами:**
- Встановити верхній LED (колір RGB)
- Встановити нижній лівий/правий LED
- Встановити кільцеві LED (8 LED по периметру)
- Вимкнути всі LED

**Звук:**
- Відтворити тон (частота, тривалість)
- Відтворити системний звук
- Зупинити звук

**Умови датчиків:**
- Якщо наближення (датчик, поріг, порівняння)
- Якщо наземний темний / світлий
- Якщо кнопка натиснута / відпущена

**Таймер:**
- Встановити період таймера (таймер 0 або 1, період у мс)

### Експорт на реального Thymio

1. Експортуйте ваш проєкт Thymio через експортер Aseba
2. Відкрийте згенерований файл `.aesl` в Aseba Studio
3. Підключіть ваш Thymio через USB
4. Натисніть **Load**, потім **Run**

- Переконайтеся, що ваш проєкт має принаймні одну кімнату з екземплярами
- Перевірте, що об'єктам призначені спрайти
- Перегляньте вивід консолі (режим налагодження F6) на наявність повідомлень про помилки

### Спрайти не відображаються

- Переконайтеся, що файл спрайту існує в папці `sprites/`
- Перевірте, що об'єкт має призначений спрайт у його властивостях
- Переконайтеся, що екземпляр має `visible = true`

### Зіткнення не працюють

- Переконайтеся, що цільовий об'єкт позначений як **Твердий**, якщо використовується зіткнення на основі твердих об'єктів
- Переконайтеся, що у вас налаштована подія **Collision With** для правильного об'єкта
- Перевірте, що екземпляри дійсно перекриваються (використовуйте режим налагодження)

### Звук не відтворюється

- Переконайтеся, що звуковий файл існує та має підтримуваний формат (WAV, OGG, MP3)
- Перевірте, що pygame.mixer ініціалізувався успішно (див. вивід консолі)
- Музичні файли відтворюються потоково з диска — переконайтеся, що шлях до файлу правильний

### Помилка експорту

- **Експорт EXE:** Переконайтеся, що PyInstaller встановлений (`pip install pyinstaller`)
- **Експорт Kivy:** Переконайтеся, що Kivy встановлений (`pip install kivy`)
- **Експорт HTML5:** Перевірте консоль на наявність помилок кодування
- Усі експорти потребують дійсного проєкту з принаймні однією кімнатою

### Проблеми з продуктивністю

- Зменшіть кількість екземплярів у кімнатах
- Використовуйте систему зіткнень з просторовою сіткою (увімкнена за замовчуванням)
- Уникайте ресурсоємних операцій у подіях Step (виконуються 60 разів на секунду)
- Використовуйте будильники для періодичних завдань замість підрахунку кадрів у Step
