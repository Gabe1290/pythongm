pyGM предлагает систему визуального программирования для лёгкой разработки игр без кода.

## Обзор

С визуальным программированием вы можете:
- Создавать игровую логику методом перетаскивания
- Соединять блоки для сложного поведения
- Разрабатывать без знаний программирования

## Редактор Blockly

### Интерфейс
1. **Палитра блоков**: Доступные блоки по категориям
2. **Рабочая область**: Здесь вы соединяете блоки
3. **Панель инструментов**: Сохранить, Загрузить, Удалить

### Категории блоков
- **Логика**: Если/Тогда, сравнения, логические значения
- **Циклы**: Повторения
- **Математика**: Вычисления
- **Текст**: Операции с текстом
- **Переменные**: Хранение значений
- **Функции**: Повторно используемые блоки
- **Игра**: Специфические действия pyGM

## Использование блоков

### Добавление блока
1. Кликните на категорию
2. Перетащите блок на рабочую область
3. Соедините его с другими блоками

### Соединение блоков
- Блоки автоматически соединяются
- Обращайте внимание на соответствующие формы
- Вложение блоков возможно

### Настройка блока
- Заполните поля ввода
- Выберите опции из выпадающего меню
- Вставьте подблоки

## Примеры

### Простое движение
```
Когда [стрелка вправо] нажата
  Установить x на (x + 5)
```

### Условная логика
```
Если <Жизни <= 0> тогда
  Показать сообщение "Game Over"
  Перейти в комнату [rm_gameover]
```

### Цикл
```
Повторить [10] раз
  Создать экземпляр [obj_moneta] на позиции (Случайно 0-800, Случайно 0-600)
```

## Игровые блоки

### Движение
- **Переместить к**: Движение к позиции
- **Установить скорость**: Скорость движения
- **Установить направление**: Направление движения

### Экземпляры
- **Создать экземпляр**: Создать новый объект
- **Уничтожить**: Удалить объект
- **Для всех**: Все экземпляры типа

### Переменные
- **Установить переменную**: Сохранить значение
- **Изменить переменную**: Изменить значение
- **Получить переменную**: Получить значение

### События
- **Когда клавиша**: Ввод с клавиатуры
- **Когда столкновение**: Контакт объектов
- **Когда таймер**: На основе времени

## Советы

1. **Начинайте с малого**: Сначала простые проекты
2. **Тестируйте**: Регулярно запускайте
3. **Организуйте**: Логически группируйте блоки
4. **Комментарии**: Добавляйте заметки
