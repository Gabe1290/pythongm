## Beginner Tutorials

These tutorials are designed for users new to game development. They use the **Beginner Preset** and cover fundamental concepts.

### Classic Games

| Tutorial | Description | Skills Learned |
|----------|-------------|----------------|
| [**Pong**](Tutorial-Pong) | Create a classic two-player Pong game | Keyboard input, collision detection, score tracking, global variables |
| [**Breakout**](Tutorial-Breakout) | Build a brick breaker game | Object destruction, bounce mechanics, lives system |
| [**Sokoban**](Tutorial-Sokoban) | Create a box-pushing puzzle game | Grid-based movement, push mechanics, win conditions |

### Comprehensive Guides

| Tutorial | Description | Skills Learned |
|----------|-------------|----------------|
| [**Introduction to Game Creation**](Getting-Started-Breakout) | A complete beginner's guide covering all PyGameMaker basics through building a Breakout game | Sprites, objects, events, actions, parent objects, rooms |

## What You'll Learn

Each tutorial teaches important game development concepts:

### Movement & Physics
- Setting object speed and direction
- Using gravity and friction
- Bouncing objects off walls and other objects

### User Input
- Handling keyboard input (press, release, hold)
- Responding to mouse clicks and movement

### Game Logic
- Using events to trigger actions
- Managing game state with variables
- Tracking scores, lives, and health

### Visual Feedback
- Drawing text and shapes
- Displaying scores and status
- Creating visual effects

## Getting Started

1. **Install PyGameMaker** - Follow the [installation instructions](Home#installation)
2. **Choose a Tutorial** - Start with [Pong](Tutorial-Pong) or [Breakout](Tutorial-Breakout)
3. **Set Your Preset** - Use the Beginner Preset for these tutorials
4. **Follow Along** - Work through each step carefully
5. **Experiment** - Try the enhancement suggestions at the end!

---

## Tips for Success

- **Read each step carefully** before implementing it
- **Test frequently** - Run your game after each major addition
- **Don't skip steps** - Each step builds on the previous one
- **Use the troubleshooting sections** if something doesn't work
- **Experiment** - Once you complete a tutorial, try modifying it!
