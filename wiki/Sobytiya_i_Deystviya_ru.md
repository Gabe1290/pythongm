События и действия формируют сердце игровой логики в pyGM.

## Концепция

### События
События - это триггеры, которые реагируют на определённые ситуации:
- Запуск игры
- Нажатие клавиш
- Столкновение
- Таймер

### Действия
Действия - это ответы на события:
- Движение
- Создание/Уничтожение
- Изменение значений
- Воспроизведение звуков

## Категории событий

### События создания
- **Create**: Один раз при создании экземпляра
- **Destroy**: При удалении экземпляра
- **Room Start**: При входе в комнату

### События Step
- **Step**: Каждый кадр
- **Begin Step**: Перед проверкой столкновений
- **End Step**: После проверки столкновений

### События ввода
- **Клавиатура**: Нажатие/отпускание клавиш
- **Мышь**: Клики и движение
- **Геймпад**: Ввод с контроллера

### События столкновения
- Контакт с другими объектами
- Контакт со стенами
- Проверки областей

### События рисования
- **Draw**: Обычное рисование
- **Draw GUI**: Элементы интерфейса
- **Draw Begin/End**: До/После рисования

### Другие события
- **Alarm**: События на основе таймера
- **Animation End**: Анимация спрайта завершена
- **User Events**: Пользовательские события

## Библиотека действий

### Движение
- `move_towards`: Движение к точке
- `set_speed`: Установить скорость
- `set_direction`: Установить направление
- `bounce`: Отскочить

### Экземпляры
- `instance_create`: Создать новый экземпляр
- `instance_destroy`: Удалить экземпляр
- `change_sprite`: Сменить спрайт

### Переменные
- `set_variable`: Установить значение
- `add_to_variable`: Добавить значение
- `if_variable`: Условная проверка

### Аудио
- `play_sound`: Воспроизвести звук
- `stop_sound`: Остановить звук
- `set_volume`: Изменить громкость

### Комната
- `goto_room`: Сменить комнату
- `restart_room`: Перезапустить комнату
- `goto_next_room`: Следующая комната

### Рисование
- `draw_sprite`: Нарисовать спрайт
- `draw_text`: Показать текст
- `draw_rectangle`: Нарисовать прямоугольник

## Условия и управление потоком

### Условные действия
```
Если Переменная == Значение
  Выполнить действие
Иначе
  Альтернативное действие
```

### Циклы
- Повторить действия
- Для всех экземпляров

## Лучшие практики

1. **Используйте Step умеренно**: Только когда необходимо
2. **Оптимизируйте столкновения**: Учитывайте свойство Solid
3. **Группируйте события**: Связанная логика вместе
4. **Используйте аларми**: Для действий по времени
